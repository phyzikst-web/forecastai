1. 시계열 예측 문제는 통계 문제만이 아닙니다

딥러닝 시계열 모델은 과거 관측값을 잘 맞추면서도 물리적으로 불가능한 예측을 만들 수 있습니다. 강수량이 음수가 되거나, 방사성 붕괴량이 증가하거나, 닫힌 계에서 에너지가 생성되거나, 저수지 저장량이 용량을 초과하는 식입니다. 문제는 신경망의 표현력이 부족해서가 아닙니다. 오히려 너무 자유롭기 때문에 생깁니다. 데이터가 적거나, 잡음이 크거나, 학습 분포 밖으로 예측해야 할 때 경험적 위험 최소화는 검증 오차가 낮지만 생성 메커니즘을 어기는 함수를 선택할 수 있습니다.

물리 정보 신경망(Physics-Informed Neural Networks, PINNs)은 관측 데이터뿐 아니라 지배 방정식의 잔차까지 학습 목적에 넣는 방식으로 이 문제를 명확히 제기했습니다(Raissi et al., 2019). 더 넓게는 theory-guided data science가 과학적 일관성을 이용해 가설 공간을 제한하는 접근을 정리했습니다(Karpatne et al., 2017). 시계열 예측에서 실제 질문은 조금 더 구체적입니다. 알려진 동역학을 신경망에 어떻게 넣어야 하며, 모든 페널티 항을 hard constraint처럼 오해하지 않으려면 무엇을 구분해야 할까요?

이 구분은 중요합니다. 손실 함수의 페널티는 위반을 억제하지만 위반을 불가능하게 만들지는 않습니다. Hard constraint는 재매개변수화, 투영, 또는 제약 최적화를 통해 만족되어야 합니다. 많은 예측 시스템에서는 두 접근을 함께 쓰는 편이 좋습니다. 물리 잔차 손실로 학습 방향을 잡고, 절대 깨지면 안 되는 조건은 구조적으로 만족하도록 설계하는 방식입니다.

2. 데이터 손실에서 물리 잔차로

모델 \(f_\theta\)가 과거 창 \(\mathbf{x}_{t-L+1:t} \in \mathbb{R}^{L \times d}\)를 받아 예측 구간 \(\hat{\mathbf{y}}_{t+1:t+H} \in \mathbb{R}^{H \times m}\)을 출력한다고 하겠습니다. 일반적인 지도학습 목적함수는 다음과 같습니다.

\[ \mathcal{L}_{data}(\theta) = \frac{1}{N}\sum_{i=1}^{N} \left\| \hat{\mathbf{y}}^{(i)}_{t+1:t+H} - \mathbf{y}^{(i)}_{t+1:t+H} \right\|_2^2 . \]

여기서 \(N\)은 학습 창의 개수, \(L\)은 lookback 길이, \(d\)는 입력 변수 수, \(H\)는 예측 horizon, \(m\)은 목표 변수 수입니다. 이 목적함수는 출력이 라벨과 가까운지만 묻습니다. \(\hat{\mathbf{y}}_t\)에서 \(\hat{\mathbf{y}}_{t+1}\)로 넘어가는 과정이 질량 보존, 단조성, 경계 조건, 미분 방정식을 만족하는지는 묻지 않습니다.

스칼라 상태 \(y(t)\)가 근사적으로 다음 감쇠 법칙을 따른다고 해보겠습니다.

\[ \frac{dy(t)}{dt} = -k y(t), \qquad k > 0. \]

시간 간격이 \(\Delta t\)인 이산 예측에서는 잔차를 다음처럼 쓸 수 있습니다.

\[ r_{t+1}(\theta) = \frac{\hat{y}_{t+1} - y_t}{\Delta t} + k y_t . \]

그러면 물리 항을 더한 목적함수는 다음과 같습니다.

\[ \mathcal{L}(\theta) = \mathcal{L}_{data}(\theta) + \lambda_{phys} \frac{1}{N}\sum_{i=1}^{N} \left\|r^{(i)}_{t+1}(\theta)\right\|_2^2 . \]

\(\lambda_{phys}\)는 데이터 적합과 물리 일관성 사이의 trade-off를 조절합니다. 너무 작으면 물리 항은 장식에 가깝고, 너무 크면 실제 편차, 관측 오차, 외부 forcing, 누락된 과정까지 억지로 지워버릴 수 있습니다. 연속시간 PINN이나 neural differential model에서 이런 구성이 편리한 이유는 자동미분 때문입니다. 시간이나 상태에 대한 모델 출력의 도함수를 아키텍처마다 손으로 유도하지 않고 학습 그래프 안에서 계산할 수 있습니다(Baydin et al., 2018).

3. 무엇이 hard이고, 무엇이 soft인가요?

가장 흔한 구현은 손실 함수에 페널티 항을 추가하는 방식입니다. 이것은 soft constraint입니다.

\[ \mathcal{L}_{soft} = \mathcal{L}_{data} + \lambda \mathcal{L}_{constraint}. \]

이 방식은 표본화된 학습 지점에서 평균적인 위반을 줄입니다. 하지만 미래 예측이 반드시 법칙을 만족한다고 보장하지 않습니다. Hard constraint에는 다른 장치가 필요합니다.

제약 유형 예시 부과 방법 보장 수준
잔차 페널티 ODE/PDE 잔차, 보존 오차 손실에 \(\lambda \|r_\theta\|^2\) 추가 정확한 보장 없음
Barrier 또는 hinge 페널티 용량 \(0 \le y \le C\) \(\max(0, y-C)\) 또는 \(\max(0, -y)\) 페널티 더 강한 억제, 그래도 정확하지 않음
재매개변수화 \(y_{t+1}=y_t\exp(-\operatorname{softplus}(g_\theta)\Delta t)\) 출력 형태 자체를 제약에 맞게 정의 인코딩한 성질은 정확히 만족
투영 feasible set으로 예측값 보정 예측 후 가능한 영역으로 projection 투영이 맞으면 정확히 만족
제약 최적화 등식/부등식 제약 포함 optimizer가 제약을 직접 처리 solver tolerance 내에서 만족

운영 예측에서는 비음수성, 단조 감쇠, 재고 보존, 알려진 상한과 하한 같은 조건이 hard treatment에 적합한 경우가 많습니다. 반면 근사 법칙, 불확실한 계수, 해상도 아래의 미해결 과정은 잔차 손실로 다루는 편이 안전합니다. 정확히 만족시키려다가 모델이 자신 있게 틀릴 수 있기 때문입니다.

4. 작은 재현 예제

아래 코드는 잡음이 있는 지수 감쇠 합성 데이터로 만든 예제입니다. 세 가지 one-step forecaster를 비교합니다. 첫 번째는 무제약 differentiable basis model, 두 번째는 물리 잔차 손실을 더한 같은 모델, 세 번째는 출력이 양수 단조 감쇠 형태가 되도록 재매개변수화한 모델입니다. 예제는 Quarto의 가벼운 실행 환경에서도 렌더링되도록 numpy만 사용했습니다. LSTM, temporal convolution, Transformer에서도 같은 잔차 항을 학습 목적함수에 더하면 원리는 같습니다.

import numpy as np

seed = 7
rng = np.random.default_rng(seed)

dt = 0.25
k_true = 0.23
n_steps = 96
t = np.arange(n_steps + 1, dtype=float) * dt
y_clean = 5.0 * np.exp(-k_true * t)
y_obs = np.maximum(y_clean + rng.normal(0.0, 0.035, size=y_clean.shape), 1e-3)

train_end = 64
x_all = np.column_stack([
    np.ones(n_steps),
    t[:-1],
    y_obs[:-1],
    t[:-1] * y_obs[:-1],
    y_obs[:-1] ** 2,
])
y_next_all = y_obs[1:]
y_now_all = y_obs[:-1]

X_train, X_test = x_all[:train_end], x_all[train_end:]
y_train, y_test = y_next_all[:train_end], y_next_all[train_end:]
y_now_train, y_now_test = y_now_all[:train_end], y_now_all[train_end:]

def ridge_solution(X, y, alpha=1e-6):
    p = X.shape[1]
    return np.linalg.solve(X.T @ X + alpha * np.eye(p), X.T @ y)

def physics_residual_solution(X, y_next, y_now, lam=0.5, alpha=1e-6):
    p = X.shape[1]
    A = X.T @ X + lam * (X.T @ X) / (dt ** 2) + alpha * np.eye(p)
    b = X.T @ y_next + lam * X.T @ (y_now / dt - k_true * y_now) / dt
    return np.linalg.solve(A, b)

def evaluate(name, pred, y_test, y_now_test):
    rmse = np.sqrt(np.mean((pred - y_test) ** 2))
    violation = np.mean(pred > y_now_test)
    negative = np.mean(pred < 0)
    residual_mae = np.mean(np.abs((pred - y_now_test) / dt + k_true * y_now_test))
    return name, rmse, violation, negative, residual_mae

w_free = ridge_solution(X_train, y_train)
w_phys = physics_residual_solution(X_train, y_train, y_now_train, lam=0.5)

# Hard parameterization: fit a positive decay rate from the training transitions.
ratios = np.clip(y_train / y_now_train, 1e-6, 1.0)
k_hat = np.maximum(0.0, -np.mean(np.log(ratios)) / dt)

results = [
    evaluate("unconstrained_basis_model", X_test @ w_free, y_test, y_now_test),
    evaluate("physics_residual_loss", X_test @ w_phys, y_test, y_now_test),
    evaluate(
        "hard_decay_parameterization",
        y_now_test * np.exp(-k_hat * dt),
        y_test,
        y_now_test,
    ),
]

print("model,rmse,monotonicity_violation_rate,negative_prediction_rate,ode_residual_mae")
for row in results:
    print(f"{row[0]},{row[1]:.4f},{row[2]:.4f},{row[3]:.4f},{row[4]:.4f}")
model,rmse,monotonicity_violation_rate,negative_prediction_rate,ode_residual_mae
unconstrained_basis_model,0.1931,0.0625,0.7500,0.6713
physics_residual_loss,0.0402,0.0312,0.1875,0.0746
hard_decay_parameterization,0.0385,0.0000,0.0000,0.0047

이 합성 데이터 실행에서는 잔차 손실 모델이 무제약 basis model보다 ODE 잔차를 줄입니다. Hard parameterization 모델은 예측 형태가 \(y_t\exp(-\hat{k}\Delta t)\)이고 \(\hat{k} \ge 0\)이므로 단조성 위반을 완전히 제거합니다. 더 큰 신경망에서는 \(k\)를 비음수 네트워크 출력 \(\hat{k}_t=\operatorname{softplus}(g_\theta(\mathbf{x}_t))\)로 바꿀 수 있습니다. 핵심은 그대로입니다. 물리적 타당성과 라벨 적합은 관련되어 있지만 같은 목표가 아닙니다.

5. 제약 가중치를 어떻게 고를 것인가요?

물리 정보를 넣은 학습에서 가장 어려운 부분은 잔차를 쓰는 일이 아닙니다. 잔차가 얼마나 중요해야 하는지 정하는 일입니다. 고정된 \(\lambda_{phys}\)는 단순하지만 단위 불일치를 감춥니다. 데이터 손실은 섭씨 온도의 제곱일 수 있고, 보존 잔차는 초당 킬로그램 단위일 수 있습니다. 원래 물리량의 스케일로 각 항을 정규화하는 편이 raw magnitude를 직접 튜닝하는 것보다 안전합니다.

실무적으로는 세 가지를 확인해야 합니다. 첫째, train, validation, test 기간의 잔차 분포를 비교합니다. Collocation point에서만 물리를 만족하는 모델은 다른 지점에서 실패할 수 있습니다. 둘째, 하나의 가중치 결과만 보고하지 말고 \(\mathcal{L}_{data}\)와 \(\mathcal{L}_{phys}\) 사이의 Pareto curve를 확인합니다. 셋째, 더 긴 horizon, 드문 forcing, 경계값, 누락된 공변량처럼 외삽에 가까운 구간을 따로 시험합니다. Scientific ML 리뷰들은 물리 지식이 데이터 효율성을 높일 수 있다고 설명하지만, 그 효과는 인코딩한 법칙이 해당 regime에 맞고 최적화 문제가 잘 조건화되어 있을 때에 달려 있습니다(Karniadakis et al., 2021; Willard et al., 2022).

6. Hard constraint를 쓰지 말아야 할 때

Hard constraint는 법칙이 불완전하거나, 관측 과정이 상태 방정식과 다르거나, 외생 forcing이 빠져 있을 때 위험합니다. 예를 들어 저수지 저장량 모델은 강수, 방류, 증발, 침투, 관측 보정까지 고려한 뒤에야 물 보존을 만족해야 합니다. 열린 시스템에 닫힌 시스템의 질량 보존을 강제하면 일관성은 아름답지만 편향된 예측기가 됩니다.

기상·환경 AI에서도 같은 주의가 필요합니다. 많은 제약은 해상도에 의존합니다. 격자 규모의 PDE 잔차가 관측소 자료를 직접 설명하려면 representativeness model이 필요할 수 있습니다. 단조성 규칙은 누적량에는 맞지만 잡음이 있는 센서값에는 맞지 않을 수 있습니다. 이런 경우에는 정확한 규칙보다 허용 오차가 있는 잔차나 확률적 잔차를 손실 함수에 넣는 편이 낫습니다.

7. 실제 적용 순서

먼저 제약을 목표 변수의 단위로 씁니다. 그다음 위반이 절대 허용되지 않는지, 아니면 드물지만 가능한지 결정합니다. 절대 허용되지 않는다면 재매개변수화, 투영, 제약 decoding을 우선 고려합니다. 법칙이 근사적이라면 잔차 손실을 쓰고, forecast error와 residual error를 함께 보면서 \(\lambda_{phys}\)를 조정합니다. 시계열 분할은 시간 순서를 유지해야 하며, 잔차 계산에는 예측 시점에 사용할 수 있는 정보만 넣어야 합니다. 마지막으로 제약이 도움이 되는 조건과 오히려 해로운 조건을 모두 보고해야 합니다.

도메인 지식의 가치는 딥러닝을 덜 경험적으로 만드는 데 있지 않습니다. 경험적 문제를 더 정직하게 만드는 데 있습니다. 정확하지만 물리적으로 불가능한 예측은 배포 가능한 과학 모델이 아닙니다. 반대로 물리적으로 우아하지만 데이터를 무시하는 예측도 쓸모가 없습니다. 실제 목표는 통계적 적합, 물리적 일관성, 운영상 허용 범위가 만나는 좁은 영역을 찾는 것입니다.
