Recursive, Direct, Joint: 멀티 스텝 예측 전략은 무엇을 포기하는가

데이터과학
인공지능
시계열

H-스텝 예측을 어떻게 만들어 내는지는 그 자체로 하나의 모델링 결정입니다. 재귀(Recursive), 직접(Direct), 결합(Joint, MIMO) 전략을 오차 누적, 편향-분산 트레이드오프, 계산 비용, 그리고 각 전략이 유리해지는 조건의 관점에서 비교합니다.

Prof. Shin (phyzik)

공개

2026년 6월 30일

1 “앞으로 24시간을 예측하라”는 말 속에 숨은 질문

시계열 예측 입문은 대부분 한 스텝 앞을 예측하는 모델을 학습시키는 데서 멈춥니다. 그러나 실무는 값 하나를 원하는 경우가 거의 없습니다. 전력 운영자는 향후 24시간의 부하를, 유통업체는 다음 6주의 수요를, 임상 모델은 앞으로 며칠간의 병상 점유율을 필요로 합니다. 예측 구간 \(H\)가 1을 넘는 순간, 한 스텝 설정에서는 마주칠 일이 없던 질문이 생깁니다. 과거에서 미래로의 사상(mapping)을 학습하는 모델로부터 어떻게 \(H\)개의 값을 만들어 낼 것인가?

이것은 하이퍼파라미터 수준의 세부 사항이 아닙니다. 어떤 전략을 고르느냐에 따라 모델이 실제로 최소화하는 손실이 바뀌고, 예측 오차가 구간을 따라 어떻게 누적되는지가 달라지며, 학습·저장해야 할 모델 수가 달라집니다. 또한 \(\hat{y}_{N+1}\)과 \(\hat{y}_{N+5}\) 사이의 의존 구조를 보존하느냐 버리느냐도 결정됩니다. 실무에서 지배적인 전략은 세 가지입니다. 재귀(Recursive), 직접(Direct), 그리고 결합(Joint) — 마지막 전략은 문헌에서 보통 다중 출력(Multiple-Output) 또는 MIMO로 불립니다. 세 전략은 서로 다른 가정을 담고 있으며, 어느 것이 최선인가에 대한 실증 기록은 일관성이 없기로 유명합니다. @taieb2012 는 기존 문헌의 결론들이 서로 모순된다는 점을 지적했습니다. 어떤 연구는 재귀를, 어떤 연구는 직접을, 또 어떤 연구는 다중 출력을 지지했습니다.

이 글 전체에서 멀티 스텝 과제란 \(N\)개 관측치로 이루어진 시계열 \([y_1, \dots, y_N]\)의 다음 \(H\)개 값 \([y_{N+1}, \dots, y_{N+H}]\)를 예측하는 것을 뜻하며, \(H > 1\)은 예측 구간, \(d\)는 입력으로 쓰는 과거 시차의 개수(임베딩 차원)입니다.

2 재귀(Recursive): 하나의 모델에 자기 출력을 다시 먹이다

재귀(또는 반복, iterated) 전략은 가장 오래되고 가장 직관적입니다. 한 스텝 모델 \(f\) 하나를 학습합니다.

\[ y_{t+1} = f(y_t, y_{t-1}, \dots, y_{t-d+1}) + w_t, \tag{1}\]

여기서 \(y_t \in \mathbb{R}\)는 시점 \(t\)의 값, 입력은 가장 최근 \(d\)개 시차로 이루어진 벡터 \((y_t, \dots, y_{t-d+1}) \in \mathbb{R}^{d}\), \(w_t\)는 잡음 항이며 \(f: \mathbb{R}^d \to \mathbb{R}\)는 학습된 사상입니다. 모델은 오직 한 스텝 목적함수로만 학습됩니다.

\(H\) 스텝을 예측할 때는 \(\hat{f}\)를 반복 적용하되, 각 예측값을 관측치인 것처럼 다시 입력으로 넣습니다.

\[ \hat{y}_{N+h} = \begin{cases} \hat{f}(y_N, \dots, y_{N-d+1}) & h = 1 \\[4pt] \hat{f}(\hat{y}_{N+h-1}, \dots, \hat{y}_{N+1}, y_N, \dots, y_{N-d+h}) & 2 \le h \le d \\[4pt] \hat{f}(\hat{y}_{N+h-1}, \dots, \hat{y}_{N+h-d}) & h > d \end{cases} \tag{2}\]

핵심은 가운데 경우입니다. \(h \ge 2\)가 되는 즉시 입력 중 적어도 하나는 관측치가 아니라 예측값이 됩니다. \(h > d\)에 이르면 모든 입력이 예측값입니다. 잘 알려진 약점이 여기서 나옵니다. 오차가 앞으로 전파됩니다. 1스텝의 작은 실수가 2스텝의 입력 일부가 되어 더 큰 실수를 낳고, 그 과정이 반복됩니다. 모델은 자신의 잡음 섞인 출력으로 학습된 적이 없으므로 그에 강건해질 기회가 없습니다. 학습 입력 분포와 예측 입력 분포가 어긋나는 것입니다.

이를 뒷받침하는 이론적 진술은, 한 스텝 오차의 최소화가 \(H\) 스텝 오차의 최소화를 보장하지 않으며 재귀 전략은 점근적으로 편향(asymptotically biased)된다는 것입니다 [@taieb_rectify]. 장점도 그만큼 분명합니다. 모델이 하나뿐이라 저장 비용이 낮고, 그 단일 모델에 전체 학습 데이터를 쓸 수 있으며, 재학습 없이 임의의 구간 \(H\)까지 도달할 수 있습니다.

3 직접(Direct): 구간마다 모델 하나씩

직접 전략은 각 스텝마다 별도의 모델을 학습시켜 편향 문제를 정면으로 공략합니다. 각 \(h \in \{1, \dots, H\}\)에 대해 \(f_h\)를 적합합니다.

\[ y_{t+h} = f_h(y_t, y_{t-1}, \dots, y_{t-d+1}) + w_t, \tag{3}\]

따라서 \(f_h: \mathbb{R}^d \to \mathbb{R}\)는 동일한 \(d\)개의 관측 시차를 \(h\) 스텝 앞의 값으로 직접 사상합니다. 예측은 피드백 없이 한 번에 산출됩니다.

\[ \hat{y}_{N+h} = \hat{f}_h(y_N, y_{N-1}, \dots, y_{N-d+1}), \quad h = 1, \dots, H. \tag{4}\]

모든 \(\hat{f}_h\)가 관측된 값만 입력으로 받으므로 오차 누적이 없습니다. 재귀 전략의 실패 양상이 구조적으로 부재합니다. 또 각 모델은 자기 구간에 대해 정확히 우리가 원하는 목적함수를 최적화하므로, 직접 전략은 이론적으로 비편향(unbiased)입니다 [@taieb_rectify].

비용은 세 군ده에서 나타납니다. 첫째, 모델을 하나가 아니라 \(H\)개 학습·저장해야 하므로 \(H\)가 크면 부담이 됩니다. 둘째, 각 모델이 독립적으로 학습되므로 예측들이 구간 간 의존 구조를 무시합니다. \(\hat{y}_{N+5}\)와 \(\hat{y}_{N+4}\)를 묶어 주는 것이 없어 들쭉날쭉하고 서로 모순되는 궤적이 나올 수 있습니다. 셋째, 구간이 길수록 유효 학습 표본 수가 줄며, 비편향이라고 해서 유한 표본에서 일치성(consistency)이 보장되는 것은 아닙니다 [@taieb_rectify].

4 결합(Joint, MIMO): 하나의 모델, 벡터 출력

결합 전략 — 다중 입력 다중 출력, MIMO — 은 전체 구간을 한 번에 내보내는 단일 모델을 학습합니다. 스칼라 대신 \(F\)가 벡터를 반환합니다.

\[ [y_{t+1}, \dots, y_{t+H}] = F(y_t, y_{t-1}, \dots, y_{t-d+1}) + \mathbf{w}_t, \tag{5}\]

이제 \(F: \mathbb{R}^d \to \mathbb{R}^H\)는 \(d\)개의 입력 시차를 단 한 번의 연산으로 \(H\)차원 출력 벡터에 사상하며, \(\mathbf{w}_t \in \mathbb{R}^H\)는 벡터 잡음 항입니다. 예측은 단순히 다음과 같습니다.

\[ [\hat{y}_{N+1}, \dots, \hat{y}_{N+H}] = \hat{F}(y_N, \dots, y_{N-d+1}). \tag{6}\]

두 가지 성질이 따라옵니다. 재귀가 없으므로 오차 누적이 없습니다. 그리고 \(H\)개 출력이 모두 매개변수를 공유하는 하나의 모델에서 나오므로, 이 전략은 예측값들 사이의 확률적 의존 구조를 보존합니다. 애초에 MIMO를 도입한 동기가 바로 이것입니다 [@taieb2012]. \(H\)개 유닛의 출력층을 가진 신경망이나 시퀀스-투-시퀀스 디코더가 현대적인 자연스러운 구현입니다.

대가는 유연성입니다. 공유된 단일 모델은 모든 구간에 동일한 함수 형태를 강제합니다. 입력 윈도우와 \(h=1\)의 관계가 \(h=H\)의 관계와 다를 수 있는데도 그렇습니다. 중간 형태인 DIRMO 전략은 바로 이 지점을 보간하기 위해 제안되었습니다. 크기 \(s\)의 블록 단위로 예측하여 \(H/s\)개의 모델이 각각 \(s\)차원 벡터를 내보내게 함으로써, 의존 구조 보존과 구간별 유연성 사이에서 절충합니다 [@taieb2012].

표는 핵심 긴장을 드러냅니다. 재귀와 결합은 모두 모델 하나를 쓰지만(저렴), 재귀는 오차 누적으로, 결합은 비유연성으로 그 대가를 치릅니다. 직접은 두 문제를 모두 제거하는 대신 모델링 비용을 \(H\)배로 늘리고 구간 간 연결을 끊습니다. 지배적인 열은 없습니다. 바로 그래서 실증 문헌이 엇갈립니다.

6 재현 가능한 비교 골격

아래 코드는 단일 시계열에서 공정한 비교를 구성합니다. 의도적으로 전략만 달라지도록 단순한 회귀기를 씁니다. 학습기가 아니라 전략이 변수가 되도록 한 것입니다. 방법론적 요점은 세 가지입니다. 학습/테스트 분할이 시간 순서를 지키고(셔플 없음, 미래 정보 누수 없음), 모든 전략에 동일한 시차 윈도우 \(d\)와 구간 \(H\)를 쓰며, 난수 시드를 고정합니다.

import numpy as np
from sklearn.linear_model import Ridge
from sklearn.metrics import mean_squared_error

SEED = 42
np.random.seed(SEED)

# 합성 시계열: 추세 + 계절성 + 잡음.
# 벤치마크 결과가 아니라 메커니즘 예시용 골격이다.
N = 600
t = np.arange(N)
series = (
    0.01 * t
    + 2.0 * np.sin(2 * np.pi * t / 24)
    + np.random.normal(0, 0.5, size=N)
)

D = 24   # 임베딩 차원(입력으로 쓰는 시차 수)
H = 12   # 예측 구간
TEST = 100
def make_supervised(s, d, h):
    """X는 (n_samples, d) 형태의 과거 시차, Y는 (n_samples, h) 형태의
    다음 h개 타깃인 (X, Y)를 만든다. 시간 순서를 보존하며,
    행 i는 자신의 윈도우 이후 정보를 절대 보지 않는다."""
    X, Y = [], []
    for i in range(d, len(s) - h + 1):
        X.append(s[i - d:i])          # 과거 d개 관측치
        Y.append(s[i:i + h])          # 미래 h개 타깃
    return np.array(X), np.array(Y)

train_series = series[:-TEST]
X_tr, Y_tr = make_supervised(train_series, D, H)
# X_tr: (n_samples, D)   Y_tr: (n_samples, H)
def fit_recursive(X, y_onestep):
    """1-스텝 타깃(Y의 첫 열)으로 학습한 단일 모델."""
    m = Ridge(alpha=1.0, random_state=SEED)
    m.fit(X, y_onestep)
    return m

def forecast_recursive(model, last_window, h, d):
    win = list(last_window)
    preds = []
    for _ in range(h):
        yhat = model.predict(np.array(win[-d:]).reshape(1, -1))[0]
        preds.append(yhat)
        win.append(yhat)             # 예측값을 다시 입력으로
    return np.array(preds)

def fit_direct(X, Y):
    """구간마다 하나씩, H개의 독립 모델."""
    return [Ridge(alpha=1.0).fit(X, Y[:, h]) for h in range(Y.shape[1])]

def forecast_direct(models, last_window):
    x = last_window.reshape(1, -1)
    return np.array([m.predict(x)[0] for m in models])

def fit_joint(X, Y):
    """단일 다중 출력 모델: Ridge는 벡터 타깃을 직접 지원한다."""
    return Ridge(alpha=1.0).fit(X, Y)

def forecast_joint(model, last_window):
    return model.predict(last_window.reshape(1, -1))[0]
rec = fit_recursive(X_tr, Y_tr[:, 0])
dir_ = fit_direct(X_tr, Y_tr)
joint = fit_joint(X_tr, Y_tr)

last = train_series[-D:]
truth = series[-TEST:-TEST + H]

p_rec   = forecast_recursive(rec, last, H, D)
p_dir   = forecast_direct(dir_, last)
p_joint = forecast_joint(joint, last)

for name, p in [("recursive", p_rec), ("direct", p_dir), ("joint", p_joint)]:
    print(name, np.sqrt(mean_squared_error(truth, p)))
recursive 0.402653629333326
direct 0.40946224616781524
joint 0.4094622461678082

평가 셀을 실행하여 아래와 같이 성능 수치(RMSE) 결과를 얻을 수 있습니다. 로컬에서 — 가능하면 단일 분할이 아니라 여러 윈도우에 대한 롤링 오리진(rolling origin) 평가로 — 실행한 뒤 채워 넣을 구조는 다음과 같습니다.

전략 지표 짧은 구간 긴 구간
재귀(Recursive) RMSE 실행 후 입력 실행 후 입력
직접(Direct) RMSE 실행 후 입력 실행 후 입력
결합(Joint, MIMO) RMSE 실행 후 입력 실행 후 입력

단일 집계 수치보다 구간의 함수로서의 오차를 보고하는 편이 더 중요합니다. 전략들은 \(h\)가 커질수록 갈라지는 경향이 있는데, 평균 RMSE 하나는 정작 비교하려는 그 거동을 가립니다.

결과를 어떻게 읽고, 문헌은 실제로 무엇을 말하는가에 대한 설명입니다.

8 한계

이 비교는 학습기를 고정한 채 전략에 관한 것입니다. 다른 기저 모델(그래디언트 부스팅, LSTM, 트랜스포머)을 쓰면 상대 순위가 바뀔 수 있고, 일부 아키텍처는 범주를 흐립니다. 시퀀스-투-시퀀스 모델은 구성상 결합이고, 자기회귀 트랜스포머 디코더는 추론 시 재귀입니다. 여기서 쓴 합성 시계열은 메커니즘을 보여 주기 위한 것이지 결론을 내리기 위한 것이 아닙니다. 실제 판정에는 여러 실제 시계열과 적절한 재표집이 필요합니다. 재귀의 오차 누적은 잡음 수준과 기저 동역학의 안정성과도 상호작용하므로, 단일 데이터셋으로는 문제를 매듭지을 수 없습니다. 끝으로, 이 전략들 중 어느 것도 예측 불확실성을 직접 다루지 않습니다. 구간 전체에 걸친 예측 구간(predictive interval)을 정량화하는 일은 전략 선택과 얽히는 별개의 문제이며 여기서는 다루지 않았습니다.

9 결론

재귀, 직접, 결합 사이의 선택은 어떤 문제를 감수할 것인가에 대한 선택입니다. 재귀는 오차 누적을 대가로 저렴함과 임의 구간 유연성을 삽니다. 직접은 \(H\)개의 모델과 끊어진 구간 간 구조를 대가로 비편향과 구간별 맞춤을 삽니다. 결합은 모든 스텝에 하나의 함수 형태를 강제하는 대가로, 구간 간 의존 구조를 존중하는 단일의 저렴한 모델을 삽니다. 이론은 이들을 느슨하게만 줄 세우고, 실증 문헌은 공공연히 모순되며, 당신의 데이터에서 이기는 전략은 가정할 것이 아니라 구간별로 측정해야 할 대상입니다. 출발점으로 삼을 기본값 하나가 필요하다면, 적절한 계절성 제거를 동반한 다중 출력 모델은 NN5 규모의 증거가 잘 뒷받침합니다. 다만 이를 정답이 아니라 넘어서야 할 기준선으로 다루십시오.
