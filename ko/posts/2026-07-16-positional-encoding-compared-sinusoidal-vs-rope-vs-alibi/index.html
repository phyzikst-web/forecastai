위치 인코딩 비교: Sinusoidal vs. RoPE vs. ALiBi

인공지능

트랜스포머
위치인코딩

셀프 어텐션은 순서에 대해 대칭이기 때문에, 모든 트랜스포머는 위치 인코딩을 통해 순서 정보를 몰래 끼워 넣습니다. 이 글은 sinusoidal, RoPE, ALiBi 세 방식을 각자가 보장하려는 성질에서 출발해 유도하고, NumPy로 실제 구현해 어텐션 점수의 모양이 어떻게 달라지는지, 그리고 학습 시 보지 못한 시퀀스 길이로 갔을 때 무엇이 무너지는지 실행 결과와 그림으로 비교합니다.

Prof. Shin

공개

2026-07-16

어텐션에 순서 정보가 필요한 이유

트랜스포머의 스케일드 닷프로덕트 어텐션은 본질적으로 집합 연산입니다. 쿼리 \(Q\), 키 \(K\), 값 \(V\)가 각각 \((n, d_k), (n, d_k), (n, d_v)\) 형태일 때

\[\operatorname{Attention}(Q, K, V) = \operatorname{softmax}\!\left(\frac{QK^{\top}}{\sqrt{d_k}}\right) V\]

의 출력은 시퀀스 축에서 토큰 순서를 바꾸면 그대로 재정렬되는 치환동변(permutation-equivariant) 연산입니다. 연산 안에서 토큰의 위치를 들여다보는 부분이 어디에도 없습니다. 자연어·시계열·코드처럼 순서가 의미를 갖는 데이터를 다루려면 어텐션에 들어가기 전이나 그 안에서 위치 신호를 명시적으로 주입해야 합니다. 그 신호를 만드는 장치가 위치 인코딩(Positional Encoding, PE) 입니다.

현재 트랜스포머에서 지배적으로 쓰이는 세 방식은 무엇을 인코딩할지에 대해 꽤 다른 선택을 합니다. Vaswani 기타 (2017년) 의 sinusoidal PE는 절대 위치를 고정된 패턴으로 만들어 토큰 임베딩에 더합니다. Su 기타 (2024년) 가 제안한 Rotary Position Embedding(RoPE)은 쿼리와 키를 회전시켜 내적 안에 상대 위치를 심습니다. Press 기타 (2022년) 의 Attention with Linear Biases(ALiBi)는 어텐션 로짓에 거리 기반 편향을 직접 더합니다. 이 세 방식의 차이는 두 축에서 실제로 드러납니다. 학습에서 보지 못한 시퀀스 길이로 확장할 때 어떻게 되는가, 그리고 “거리”의 의미를 모델이 얼마나 스스로 학습할 수 있게 두는가입니다.

위치 인코딩이 지켜야 하는 것

세 방식을 비교하기 전에, 어텐션 안에서 인코딩된 위치가 어떤 역할을 해야 하는지 정리해 두면 도움이 됩니다. 단일 헤드에서 위치 \(m\)의 쿼리와 위치 \(n\)의 키 사이의 소프트맥스 이전 로짓은

\[s_{m,n} = \frac{ q(x_m)^{\top} k(x_n) }{\sqrt{d_k}}\]

입니다. 가산형 PE는 \(q(x_m) = W_Q(x_m + p_m)\), \(k(x_n) = W_K(x_n + p_n)\)로 정의됩니다. 내적을 전개하면

\[q(x_m)^{\top}k(x_n) = x_m^{\top} W_Q^{\top} W_K x_n + x_m^{\top} W_Q^{\top} W_K p_n + p_m^{\top} W_Q^{\top} W_K x_n + p_m^{\top} W_Q^{\top} W_K p_n\]

이 됩니다. 오직 첫 항만 순수한 내용(content) 항이며, 나머지 세 항은 내용과 위치를 섞고, 마지막 항은 순수한 위치 항입니다. 모델이 “12번과 15번은 3만큼 떨어져 있다”는 상대적 구조를 잡아내려면 마지막 항이 \((m - n)\)만의 함수여야 합니다. sinusoidal PE는 삼각함수의 합·차 공식 덕분에 이를 근사적으로 만족시키지만, 앞의 교차항이 절대 위치 정보를 흘려 넣기 때문에 완전한 병진 불변성은 갖지 못합니다. RoPE는 가산 방식을 아예 버리고 쿼리와 키를 회전시켜, 내적이 정확히 \((m - n)\)에만 의존하도록 강제합니다. ALiBi는 인코딩과 내용 내적을 억지로 맞추려는 시도를 포기하고, 로짓 \(s_{m,n}\)에 헤드별로 거리에 따른 편향을 그대로 더합니다.

세 방식은 결국 서로 다른 질문에 답합니다. sinusoidal PE는 “각 위치에 어떤 표지를 붙일 것인가”를, RoPE는 “쿼리·키의 좌표계를 어떻게 회전시켜야 그 내적이 상대 위치를 담게 되는가”를, ALiBi는 “내용은 건드리지 않고 먼 토큰에 어떤 페널티를 부과할 것인가”를 묻습니다.

Sinusoidal PE — 유도와 시각화

원래 방식은 위치 \(m\)의 \(d_\text{model}\)차원 벡터 \(p_m\)을 다음 고정 패턴으로 채웁니다.

\[p_{m, 2k} = \sin\!\bigl( m \, \theta_k \bigr), \quad p_{m, 2k+1} = \cos\!\bigl( m \, \theta_k \bigr), \quad \theta_k = \text{base}^{-2k/d_\text{model}},\]

여기서 base = 10000, \(k = 0, 1, \dots, d_\text{model}/2 - 1\)입니다. 각 차원 쌍 \((2k, 2k+1)\)는 주파수 \(\theta_k\)로 회전하며, \(d_\text{model}=512\)일 때 임베딩 전체에 걸쳐 대략 7배수의 주파수 범위가 펼쳐집니다. 두 가지 성질이 자연스럽게 따라옵니다. 첫째, 노름 \(\lVert p_m \rVert = \sqrt{d_\text{model}/2}\)가 모든 \(m\)에 대해 상수이므로, \(p_m\)을 토큰 임베딩에 더해도 벡터 길이가 왜곡되지 않습니다. 둘째, \(\sin(a + b), \cos(a + b)\)가 \(\sin b, \cos b\)에 대해 계수가 \(a\)에 의존하는 선형 결합으로 표현되므로, 어떤 고정 선형 사상 \(R_a\)가 존재해 \(p_{m+a} = R_a\, p_m\)이 성립합니다. sinusoidal PE가 “상대 위치를 담는다”는 말은 이 의미입니다. 즉 위치 사이의 사상이 학습 없이 정해집니다.

아래 코드는 \(d_\text{model} = 64,\ \text{max\_len} = 128\)에서 이 행렬을 만들고 노름이 실제로 상수임을 확인합니다.

import numpy as np
import matplotlib.pyplot as plt

def sinusoidal_pe(max_len: int, d_model: int, base: float = 10000.0) -> np.ndarray:
    pos = np.arange(max_len)[:, None].astype(np.float64)
    i   = np.arange(d_model // 2)
    inv_freq = 1.0 / (base ** (2 * i / d_model))
    pe = np.zeros((max_len, d_model), dtype=np.float64)
    pe[:, 0::2] = np.sin(pos * inv_freq)
    pe[:, 1::2] = np.cos(pos * inv_freq)
    return pe

PE = sinusoidal_pe(max_len=128, d_model=64)
print("shape:", PE.shape)
print("row norms (constant):", np.round(np.linalg.norm(PE, axis=1)[:5], 4))

내적은 거리가 멀어질수록 대체로 감소하지만 단조롭게 감소하지는 않습니다. 고주파 차원이 빠르게 진동하기 때문에 아주 멀리 떨어진 위치들끼리도 우연히 같은 방향으로 정렬될 수 있습니다. 이 비단조성은 장거리 태스크에서 실질적인 약점이 됩니다. sinusoidal의 원시 유사도만으로는 “가까운 것”이 “먼 것”보다 항상 높은 점수를 받도록 보장되지 않기 때문입니다. 행렬 전체를 히트맵으로 그려 보면 주파수 구조가 눈에 들어옵니다.

fig, ax = plt.subplots(figsize=(6.5, 3.4))

ax.set_xlabel('embedding dim')

그림을 열별로 읽으면 됩니다. 인덱스가 작은 왼쪽 열들은 모든 위치에 걸쳐 거의 균일하게 진동합니다. 고주파 채널들입니다. 인덱스가 큰 오른쪽 열들은 아주 느리게 변하며 사실상 거친 위치 카운터 역할을 합니다. 이 다중 스케일 배치가 바로, 하나의 고정된 패턴이 가까운 위치를 구분하는 동시에 수백 스텝 떨어진 위치에도 서로 다른 값을 부여할 수 있는 이유입니다. 그리고 이 그림은 실패 지점도 함께 보여 줍니다. 학습 시 사용한 길이를 넘어서면, 느린 열들은 모델이 한 번도 본 적 없는 값의 영역으로 계속 흘러갑니다. 외삽이 공짜가 아니라는 뜻입니다.

RoPE — 더하지 말고 회전시키자

RoPE는 “위치를 회전으로 본다”는 직관을 그대로 밀어붙입니다. 쿼리 벡터 \(q \in \mathbb{R}^{d_k}\)를 \(d_k / 2\)개의 2차원 쌍으로 쪼갠 뒤, 위치 \(m\)에서 \(k\)번째 쌍을 각 \(m \theta_k\)만큼 회전시킵니다.

\[R_{m,k} = \begin{bmatrix} \cos m\theta_k & -\sin m\theta_k \\ \sin m\theta_k & \cos m\theta_k \end{bmatrix}.\]

회전된 쿼리는 \(\tilde{q}_m = R_m q\)이고, 위치 \(n\)의 키에도 같은 연산이 적용됩니다. 이때 내적은 다음을 만족합니다.

\[\tilde{q}_m^{\top} \tilde{k}_n = q^{\top} R_m^{\top} R_n k = q^{\top} R_{n-m} k.\]

같은 쌍에 적용된 회전들은 각도가 더해지는 방식으로 합성되기 때문입니다. 어텐션 로짓이 오로지 \((n - m)\)에만 의존하게 되고, 가산 임베딩도 학습 파라미터도 없이 상대 위치가 내적 안에 그대로 들어갑니다. 결정적으로 RoPE는 쿼리와 키 계산의 내부에 놓이며 값 벡터는 건드리지 않습니다. 이 덕분에 내용과의 상호작용이 곱셈적으로 이뤄지고, 가산형 PE의 4항 전개에서 생기는 교차항 문제가 사라집니다.

최소한의 NumPy 구현으로 이 연산과 그 보장을 눈으로 확인할 수 있습니다.

def apply_rope(x: np.ndarray, base: float = 10000.0) -> np.ndarray:
    """x: (seq_len, d), d even. Rotate each (x_{2k}, x_{2k+1}) pair by m*theta_k."""
    seq_len, d = x.shape
    assert d % 2 == 0
    k     = np.arange(d // 2)
    theta = base ** (-2 * k / d)               # (d/2,)
    m     = np.arange(seq_len)[:, None]        # (seq, 1)
    ang   = m * theta[None, :]                 # (seq, d/2)
    cos, sin = np.cos(ang), np.sin(ang)
    x_even, x_odd = x[:, 0::2], x[:, 1::2]
    out = np.empty_like(x)
    out[:, 0::2] = x_even * cos - x_odd * sin
    out[:, 1::2] = x_even * sin + x_odd * cos
    return out

# 병진 불변성: <RoPE(q,m), RoPE(k,n)>이 (n-m)에만 의존함을 확인.
rng = np.random.default_rng(0)
q = rng.standard_normal((1, 32))
k = rng.standard_normal((1, 32))
for m, n in [(0, 4), (10, 14), (30, 34), (100, 104)]:
    q_at_m = apply_rope(np.tile(q, (m + 1, 1)))[m:m+1]
    k_at_n = apply_rope(np.tile(k, (n + 1, 1)))[n:n+1]
    print(f"<RoPE q@{m:>3}, RoPE k@{n:>3}> = {float(q_at_m @ k_at_n.T):+.6f}")

절대 위치가 서로 다른 네 쌍이지만 상대 오프셋 \(n - m = 4\)이 같다는 이유로 내적이 정확히 동일한 값을 냅니다. 우연히 얻어진 성질이 아니라 구성이 보장하는 성질입니다. \(k\)(쌍 인덱스)에 따라 달라지는 것은 회전 속도 \(\theta_k\)입니다. 인덱스가 작은 쌍은 천천히 돌아 거친 카운터, 큰 쌍은 빨리 돌아 세밀한 카운터 역할을 합니다. 가장 느린 쌍과 가장 빠른 쌍에 대해 \(\cos(m \theta_k), \sin(m \theta_k)\)를 그리면 그 범위가 눈에 들어옵니다.

positions = np.arange(0, 64)
theta_low  = 10000.0 ** (-2 * 0  / 64)
theta_high = 10000.0 ** (-2 * 30 / 64)
fig, axes = plt.subplots(1, 2, figsize=(8, 3.4))
for ax_, ang, title in zip(axes,
                           [positions * theta_low, positions * theta_high],
                           [f'k=0 (θ≈{theta_low:.3f})',
                            f'k=30 (θ≈{theta_high:.5f})']):

    ax_.plot(positions, np.sin(ang), label='sin(mθ)')

왼쪽 패널은 보이는 범위 안에서 여러 주기를 완성합니다. 근접한 몇 위치를 구분하는 데 유용한 구간입니다. 오른쪽 패널은 64 위치를 지나도 겨우 미세하게만 돌아갑니다. 수백~수천 위치 차이가 있어야 의미 있게 달라지는 구간입니다. LLM의 RoPE 구현들은 base 값을 재조정하거나 position interpolation(YaRN, NTK-aware scaling 등, Chen 기타 (2023년) 참조)을 적용해 이 곡선을 늘려 훈련 길이 밖까지 “구별 가능한” 범위를 확장하며, 그 대가로 세밀한 해상도가 압축됩니다.

ALiBi — 위치를 인코딩하지 말고 거리를 처벌하자

ALiBi는 전혀 다른 길을 택합니다. 쿼리와 키는 그대로 두고, 내용 내적을 평소처럼 계산한 뒤, 헤드마다 어텐션 로짓에 거리 기반 선형 편향을 더합니다.

\[s_{m,n} \;\leftarrow\; s_{m,n} \;-\; m_h \, |m - n|,\]

여기서 \(m_h > 0\)은 헤드 \(h\)에 고유한 기울기(slope)입니다. 위치 벡터도, 회전도, 학습 파라미터도 없습니다. 원래 ALiBi 레시피는 \(H\)개 헤드에 대해 기울기를 기하수열 \(m_h = (2^{-8/H})^{h+1}\)로 두어, 서로 다른 헤드가 서로 다른 스케일로 거리를 처벌하도록 합니다.

def alibi_slopes(n_heads: int) -> np.ndarray:
    start = 2.0 ** (-8.0 / n_heads)
    return np.array([start ** (i + 1) for i in range(n_heads)])

def alibi_bias(seq_len: int, n_heads: int) -> np.ndarray:
    slopes = alibi_slopes(n_heads)                            # (h,)
    i = np.arange(seq_len)[:, None]
    j = np.arange(seq_len)[None, :]
    dist = -np.abs(i - j).astype(np.float64)                  # (s,s), 대각선에서 0
    return slopes[:, None, None] * dist[None, :, :]           # (h,s,s)

slopes = alibi_slopes(8)
print("slopes (h=8):", np.round(slopes, 4))
B = alibi_bias(seq_len=16, n_heads=8)

편향 행렬은 쿼리·키 벡터의 값과 무관하게 항상 같은 삼각형 모양을 유지합니다. \(|m - n|\)에만 의존하기 때문입니다. 헤드마다 스케일이 다르기 때문에 기하수열 형태의 기울기가 두 자릿수 규모의 거리 스케일을 동시에 훑습니다. ALiBi의 길이 외삽은 바로 여기에서 나옵니다. 이 메커니즘은 임의의 \(|m - n|\)에 대해 정의되어 있으므로, 길이 1024로 학습한 모델에 추론 시 길이 4096을 넣더라도, 인코딩이 한 번도 보지 못한 “미탐 위치” 자체가 존재하지 않습니다.

세 방식이 만드는 어텐션의 모양

세 구성은 위치 \(m\)의 쿼리가 주변을 어떻게 바라보게 되는지에 있어서 서로 다른 결과를 냅니다. 모든 위치에서 동일한 랜덤 내용 벡터를 사용해 시뮬레이션하는 방식은 학습된 모델을 재현하기 위한 것이 아니라, 내용을 고정함으로써 위치가 만드는 편향만 뽑아내기 위한 것입니다. \(d_\text{model} = 64\), 시퀀스 길이 96에서 다음 코드가 위치 \(m = 48\)의 쿼리를 96개의 키에 대해 각 방식으로 돌립니다.

seq_len, d_model, m = 96, 64, 48
rng = np.random.default_rng(1)
content_q = rng.standard_normal((d_model,)) / np.sqrt(d_model)
content_k = rng.standard_normal((seq_len, d_model)) / np.sqrt(d_model)

# sinusoidal (가산형)
sin_PE = sinusoidal_pe(seq_len, d_model)
q_sin, K_sin = content_q + sin_PE[m], content_k + sin_PE
scores_sin = K_sin @ q_sin

# RoPE (q,k에 곱해지는 형태)
q_rope = apply_rope(np.tile(content_q, (m + 1, 1)))[m]
K_rope = apply_rope(content_k)
scores_rope = K_rope @ q_rope

# ALiBi (내용 점수 위에 편향을 더함)
raw       = content_k @ content_q
slope     = slopes[3]
bias_row  = -slope * np.abs(np.arange(seq_len) - m)
scores_alibi = raw + bias_row

def softmax(x):
    z = x - x.max(); e = np.exp(z); return e / e.sum()

attn_sin, attn_rope, attn_alibi = map(softmax, (scores_sin, scores_rope, scores_alibi))

fig, ax = plt.subplots(figsize=(7, 3.6))
positions_all = np.arange(seq_len)
ax.plot(positions_all, attn_sin,   label='sinusoidal (additive)', lw=1.6)
ax.plot(positions_all, attn_rope,  label='RoPE',                  lw=1.6)
ax.plot(positions_all, attn_alibi, label='ALiBi (head 3)',        lw=1.6)
ax.axvline(m, color='k', ls='--', lw=0.8, alpha=0.6, label=f'query m={m}')
ax.set_xlabel('key position'); ax.set_ylabel('attention weight')
ax.set_title('Attention distribution over positions with random content')
ax.legend(fontsize=8); ax.grid(alpha=0.3)
plt.tight_layout()
plt.show()

세 가지가 눈에 띕니다. sinusoidal의 어텐션은 쿼리 위치에서 강하게 첨두를 이룹니다(엔트로피 1.25). \(p_m\)이 \(q\)와 \(k\) 양쪽에 더해지므로, \(n = m\)일 때 내적이 커다란 \(\lVert p_m \rVert^2\) 항을 얹기 때문입니다. 학습되지 않은 랜덤 내용에서는 무해한 특성처럼 보이지만, 실제 학습 과정에서는 모델이 이 위치-자기유사성을 넘어서야 하는 부담을 안습니다. RoPE는 가산 항이 없기 때문에 거의 균일한 분포를 보입니다(엔트로피 4.56). 내용 신호가 지배하고 위치는 위상차를 통해 개입하는데 그 위상차가 부분적으로 상쇄되는 결과입니다. ALiBi는 내용과 무관하게 쿼리 근처에서 첨두를 이루고 밖으로 갈수록 감쇠하며, 중간 정도의 엔트로피를 갖습니다. 학습된 네트워크에서는 학습된 \(W_Q, W_K\)가 이 원시 모양을 조절하지만, 각 방식이 부과하는 편향은 모델이 결국 상쇄하거나 활용해야 할 대상 그대로 남습니다.

길이 외삽 — 세 방식을 갈라놓는 실패

세 방식의 실질적 차이는 학습 때 본 적 없는 길이의 시퀀스에서 어텐션을 계산할 때 가장 뚜렷하게 드러납니다. 고정된 내용 쌍에 대해 상대 거리에 따른 원시 어텐션 로짓을 두 시퀀스 길이(128과 384)에서 그리면, 각 방식이 자신의 편안한 구간을 벗어난 직후의 행동을 추적할 수 있습니다.

def rel_similarity_curve(scheme: str, seq_len: int, d: int = 64) -> np.ndarray:
    rng = np.random.default_rng(2)
    q = rng.standard_normal((d,)) / np.sqrt(d)
    k = rng.standard_normal((d,)) / np.sqrt(d)
    m = seq_len // 2
    positions = np.arange(seq_len)

    if scheme == 'sinusoidal':
        P = sinusoidal_pe(seq_len, d)
        return (k[None, :] + P) @ (q + P[m])
    if scheme == 'rope':
        q_at = apply_rope(np.tile(q, (m + 1, 1)))[m]
        k_at = apply_rope(np.tile(k, (seq_len, 1)))
        return k_at @ q_at
    if scheme == 'alibi':
        raw   = np.tile(k @ q, seq_len)
        slope = 0.25
        return raw - slope * np.abs(positions - m)
    raise ValueError

fig, axes = plt.subplots(1, 2, figsize=(10, 3.6))
for ax_, L in zip(axes, [128, 384]):
    x = np.arange(L) - L // 2
    for name, style in [('sinusoidal', ':'), ('rope', '-'), ('alibi', '--')]:
        ax_.plot(x, rel_similarity_curve(name, L), style, label=name, lw=1.4)
    ax_.axvline(0, color='k', lw=0.6, alpha=0.5)
    ax_.set_xlabel('relative distance (key − query)')
    ax_.set_ylabel('raw attention logit')
    ax_.set_title(f'seq_len = {L}')
    ax_.legend(fontsize=8); ax_.grid(alpha=0.3)
plt.tight_layout()
plt.show()

학습 길이(128)에서는 세 방식 모두 쿼리 근처에서 첨두를 이루는 구조적인 곡선을 만듭니다. 길이 384에서는 그림이 달라집니다. sinusoidal 곡선은 먼 거리에서 진폭이 큰 진동을 일으키기 시작합니다. 고정된 각도 패턴이 모델이 암묵적으로 캘리브레이션된 범위를 훨씬 벗어난 \(m \theta_k\) 값에서 평가되고 있기 때문입니다. RoPE는 진동 구조를 유지하지만 비슷한 문제가 있습니다. 빠른 회전 쌍은 이미 여러 주기를 추가로 완주했기 때문에, 특정 위상 관계에 의존하던 어텐션 헤드는 이 지점에서 혼란을 겪습니다. ALiBi는 셋 중 가장 깔끔합니다. 선형 페널티는 모든 거리에서 정의되어 있으며, 멀어질수록 그만큼 더 강하게 억제됩니다. 이 단조 감쇠가 바로 ALiBi가 길이 외삽 기법으로 제안된 이유입니다 (Press 기타 2022년). 다만 이후 연구들은 외삽이 실제로 쓸 만한지는 로짓이 잘 행동하는지가 아니라 다운스트림 태스크에 따라 결정된다는 점을 반복적으로 지적해 왔습니다 (Kazemnejad 기타 2023년).

셋 중 무엇을 고를 것인가

실제 선택 시 중요한 축들을 간단한 표로 정리합니다.

항목 Sinusoidal(가산형) RoPE ALiBi
작용 위치 토큰 임베딩에 더함 어텐션 내부에서 q, k에 곱함 어텐션 로짓에 더함
인코딩 대상 절대 위치 + 근사적 상대 위치 엄격한 상대 위치 순수 거리 페널티
학습 파라미터 0 0 0(기울기 규칙 고정)
내용과의 상호작용 \(W_Q, W_K\) 이후 교차항으로 내적 보존을 통해 곱셈적으로 없음
길이 외삽 학습 길이 밖에서 취약 큰 확장은 interpolation 필요 구성상 강함
채택된 모델 원래 트랜스포머, T5 계열 LLaMA, Mistral, Qwen, Gemma BLOOM, MPT, 일부 long-context 라인

실무에서 굳어진 패턴은 이렇습니다. sinusoidal PE는 학습 범위 안에서 문제가 해결되는 고정 길이 태스크에 무난한 기본값이고, 수식이 투명해 교육용으로도 좋습니다. RoPE는 상대 구조를 깔끔하게 보존하고 context length 확장 기법들 (position interpolation, YaRN, dynamic-NTK scaling)과 잘 맞물리기 때문에 오픈소스 LLM의 사실상 표준이 됐습니다. ALiBi는 표현의 유연성 일부를 포기하는 대신 가장 깨끗한 길이 외삽 프로파일을 제공하며, 배포 환경이 긴 입력을 요구하는데 재학습 예산은 없는 경우에 유리합니다. 어느 하나가 항상 우수하지는 않으며, 선택은 “모델이 위치 관계를 스스로 학습하도록 둘 것인가, 회전으로 강제할 것인가, 거리 페널티로 강제할 것인가”의 문제입니다.

한계와 남은 질문

두 가지 한계가 특히 중요합니다. 첫째, RoPE와 ALiBi가 엄격한 상대 위치만 전달한다는 사실은 진짜로 절대 위치에 의존하는 태스크(인덱스 참조, 경계 검출, 특정 검색 포맷 등)에서는 오히려 학습을 어렵게 만들 수 있습니다. 소량의 절대 신호를 다시 붙이는 하이브리드 설계들이 제안되고 있지만 트레이드오프는 아직 정리되지 않았습니다. 둘째, “길이 외삽”은 그 자체로 논쟁적인 벤치마크입니다. ALiBi의 우아한 외삽을 보이는 논문들은 대체로 자연어 텍스트의 perplexity를 측정하며 태스크 정확도가 아닌 경우가 많고, 통제된 연구들은 위치 인코딩과 무관하게 외삽된 어텐션이 추가 컨텍스트를 충분히 활용하지 못하는 경향을 보고합니다 (Kazemnejad 기타 2023년). “백만 토큰 컨텍스트”라는 주장을 볼 때 던져야 할 질문은 모델이 그 토큰들을 사용할 수 있는가이지, 단지 크래시 없이 넣을 수 있는가가 아닙니다.

시계열 도메인만 놓고 보면, sinusoidal 계열 PE는 트랜스포머 기반 예측기에서 여전히 흔합니다. 시퀀스 길이가 알려져 있고 고정되며, RevIN 같은 입력 정규화와 가산 PE가 깔끔하게 결합되기 때문입니다. RoPE는 LLM 백본을 재활용하는 장기 예측기에 등장하기 시작했습니다. ALiBi는 언어 모델링에 비해 예측 쪽에서는 채택이 뜸한 편이며, 이는 예측 벤치마크가 언어 모델링만큼 길이 외삽을 강하게 요구하지 않기 때문이기도 합니다.

결론

Sinusoidal PE, RoPE, ALiBi는 같은 아이디어의 세 변주가 아닙니다. 이들은 어텐션 안에서 “위치”가 무엇을 의미해야 하는가에 대해 서로 다른 답을 내며, 그 차이는 어텐션 행렬의 모양, 상대 구조가 구성상 얼마나 보존되는가, 그리고 학습되지 않은 길이로 갔을 때 무엇이 무너지는가에서 드러납니다. 하나만 남긴다면 가산형에서 \(s_{m,n}\)이 네 개의 항으로 분해된다는 사실을 남기고 싶습니다. 가산 PE가 왜 민감하고, RoPE와 ALiBi가 왜 각자의 방식으로 발명되었는지를 그 분해가 그대로 설명해 주기 때문입니다. 설계 시 던져야 할 질문은 “어느 것이 최선인가”가 아니라 “내 태스크가 이 셋 중 어떤 제약을 실제로 필요로 하는가”이며, 위의 실행 코드가 그 답을 자신의 실험 환경에서 확인해 보기 위한 출발점이 됩니다.
