시계열의 평균과 분산이 시간에 따라 표류하면, 과거로 학습한 모델은 스케일조차 본 적 없는 테스트 데이터를 만납니다. 이 글은 비정상성과 개념 표류를 바닥부터 설명하고, 실행 가능한 코드로 소박한 정규화에서 가역 인스턴스 정규화(RevIN)까지 쌓아 올립니다.

1. 문제: 한 세계에서 학습하고 다른 세계에서 시험받는 모델

대부분의 머신러닝은 조용한 가정 하나에 기댑니다. 학습에 쓰는 데이터와 예측에 쓰는 데이터가 같은 분포에서 나온다는 가정입니다. 시계열은 이 가정을 예사로 깨뜨립니다. 시계열이 정상(stationary)이라는 것은 평균·분산·자기상관 같은 통계적 성질이 시간에 걸쳐 일정하다는 뜻입니다. 그러나 실제 시계열은 대개 비정상(non-stationary) 입니다. 매출은 성장하고, 물가는 오르고, 센서는 표류하며, 어떤 상품이 갑자기 인기를 끌면 수준(level)이 훌쩍 뛰어오릅니다. 학습 윈도우에서 측정한 통계는 더 이상 지금 예측하려는 윈도우를 설명하지 못합니다.

이것은 예측에서 구체적이고 해로운 결과를 낳습니다. 모델을 짜기 전에 정확히 짚어 둘 가치가 있습니다. 모델이 입력 윈도우에서 미래 윈도우로의 사상을 학습하는데 그 윈도우들의 스케일이 계속 커진다면, 테스트 시점에 모델은 학습 때 본 적 없는 수치의 입력을 받게 됩니다. 내삽(interpolation)이 아니라 외삽(extrapolation)을 요구받는 것입니다. 이 현상을 세 가지 문제를 한꺼번에 가진 합성 시계열로 구체화해 봅니다. 지수 추세, 급격한 수준 이동(개념 표류의 대역), 그리고 수준에 따라 크기가 커지는 잡음(이분산성)입니다.

import numpy as np
np.random.seed(0)

def make_series(N=2400):
    """세 가지 표류 원천을 가진 합성 비정상 시계열:
    - exp(0.0028 t): 지수 추세(곱셈적 성장)
    - t=600,1200,1800의 수준 이동: 급격한 국면 변화(개념 표류)
    - sqrt(trend)로 스케일된 잡음: 이분산(시간이 갈수록 분산 증가)"""
    t = np.arange(N)
    trend = np.exp(0.0028 * t)
    season = 0.4 * np.sin(2 * np.pi * t / 48) * np.sqrt(trend)
    level = np.zeros(N)
    for cp in [600, 1200, 1800]:
        level[cp:] += np.random.uniform(2, 5) * np.sqrt(trend[cp])
    noise = 0.08 * np.random.randn(N) * trend
    return (trend + season + level + noise).astype(np.float32)

series = make_series()

# 비정상성 정량화: 시계열을 3등분해 구간별 통계를 본다.
for i, part in enumerate(np.array_split(series, 3)):
    print(f"third {i+1}: mean={part.mean():8.2f}  std={part.std():7.2f}")
third 1: mean=    5.84  std=   5.80
third 2: mean=   54.65  std=  31.71
third 3: mean=  395.70  std= 218.94

시계열을 3등분하면 세 구간의 평균과 표준편차가 크게 다릅니다. 평균은 첫 구간에서 마지막 구간으로 가며 한 자릿수 배를 넘게 커집니다. 학습 때 앞부분만 본 모델은 뒷부분이 완전히 다른 스케일에 놓인다는 사실을 알 길이 없습니다. 바로 이 실패를 정규화가 해결해야 합니다.

2. 두 실패에 이름 붙이기: 분포 이동과 개념 표류

무엇이 잘못되고 있는지를 가리키는 용어가 둘 있습니다. 이 둘을 구분해 두면 해결책을 고르는 데 도움이 됩니다.

분포 이동(distribution shift)은 상위 개념입니다. 학습과 배포 사이에서 데이터의 결합 분포가 바뀌는 것입니다. 시계열에서는 값 자체의 평균과 분산이 시간에 따라 움직이는, 주변 통계량의 변화로 가장 눈에 띄게 나타납니다 (Kim 기타 2022년). 위 코드의 지수 추세는 순수한 분포 이동입니다. 과거와 미래의 관계는 바뀌지 않았고, 스케일만 바뀌었습니다.

개념 표류(concept drift)는 더 날카롭고 더 위험한 사촌입니다. 여기서는 입력과 목표의 관계, 즉 학습하려는 함수 자체가 시간에 따라 바뀝니다. 새 경쟁자의 진입, 규제, 팬데믹으로 생긴 수준 이동은 개념 표류입니다. 변화 이전에 데이터를 만들던 규칙이 이후에는 더 이상 그 데이터를 만들지 않습니다. 최근 연구는 예측에서 이 표류가 두 얼굴을 가진다고 정식화합니다. 시간에 따라 입력 윈도우들의 공간 안에서 이동할 수도 있고(공간 내 이동, intra-space shift), 입력 윈도우와 예측 대상인 지평 윈도우 사이에서 다를 수도 있습니다(공간 간 이동, inter-space shift) (Fan 기타 2023년). 두 번째는 놓치기 쉬운데, 이것이야말로 예측을 일반적인 지도학습보다 어렵게 만드는 지점입니다. 단 하나의 학습 예제조차 룩백과 지평이라는 서로 다를 수 있는 두 분포에 걸쳐 있기 때문입니다.

정규화는 분포 이동 쪽에 맞서는 첫 번째 방어선입니다. 이 글의 나머지는 정규화 도구를 정교함의 순서대로 쌓아 올리며 각 단계를 측정합니다.

3. 첫 시도: 전역 정규화, 그리고 그것이 실패하는 이유

정형 데이터 머신러닝의 반사 반응은 표준화입니다. 평균을 빼고 표준편차로 나누되, 학습 집합 전체에서 한 번 계산합니다. 이 전역(global) 정규화는 최적화를 위해 수치 범위를 안정시키지만, 표류에는 아무것도 하지 못합니다. 고정된 평균과 분산 하나로는 평균과 분산이 움직이는 표적인 시계열을 설명할 수 없기 때문입니다. 열 배 높은 곳에 놓인 테스트 윈도우에서 학습 평균을 빼면, 모델이 학습한 적 없는 거대한 잔차가 남습니다.

이를 측정하려면 지도학습 장치가 필요합니다. 시간 순서를 지키는(셔플 없음, 누수 없음) 윈도우 구성과 작은 MLP 예측기를 준비합니다.

import torch
import torch.nn as nn
torch.manual_seed(0)

L, H = 48, 24    # 룩백 길이, 예측 지평

def make_windows(x, L, H):
    """1차원 시계열을 (X, Y)로 변환. X는 (n, L) 룩백, Y는 (n, H) 지평.
    윈도우는 시간 순서로 만들며, 윈도우 i는 미래 정보를 포함하지 않는다."""
    X, Y = [], []
    for i in range(L, len(x) - H + 1):
        X.append(x[i - L:i])
        Y.append(x[i:i + H])
    return np.stack(X), np.stack(Y)

X, Y = make_windows(series, L, H)
n = len(X)
split = int(n * 0.7)                       # 앞 70% 학습, 뒤 30% 테스트(시간순)
Xtr, Ytr, Xte, Yte = X[:split], Y[:split], X[split:], Y[split:]

class MLP(nn.Module):
    """최소 예측기: L 입력 -> 128 은닉 -> H 출력."""
    def __init__(self, L, H):
        super().__init__()
        self.net = nn.Sequential(nn.Linear(L, 128), nn.ReLU(), nn.Linear(128, H))
    def forward(self, x):
        return self.net(x)

print(f"train windows: {Xtr.shape}, test windows: {Xte.shape}")
print(f"train input mean: {Xtr.mean():.1f}, test input mean: {Xte.mean():.1f} "
      f"({Xte.mean()/Xtr.mean():.1f}x higher)")
train windows: (1630, 48), test windows: (699, 48)
train input mean: 33.8, test input mean: 385.6 (11.4x higher)

마지막 줄이 문제 전체를 숫자 하나로 보여 줍니다. 테스트 윈도우는 학습 윈도우보다 대략 한 자릿수 배 높은 수준에 놓여 있습니다. 이제 어떤 모델도, 학습 집합에 맞춘 어떤 전역 정규화도 외삽을 하는 셈입니다.

4. 작동하는 아이디어: 인스턴스마다 정규화하고, 다시 되돌리기

가역 인스턴스 정규화(Reversible Instance Normalization, RevIN)가 도입한 핵심 동작은, 전역으로 정규화하기를 멈추고 인스턴스마다 정규화하되 그 정규화를 가역으로 만드는 것입니다 (Kim 기타 2022년). 각 입력 윈도우에 대해, 모델이 보기 전에 그 윈도우 자신의 평균과 표준편차를 제거합니다. 그러면 어떤 절대 수준에서 왔든 모든 윈도우가 정상처럼 보이는 상태로 모델에 도착합니다. 그다음 모델이 (정규화된) 예측을 내면, 그 윈도우의 통계를 다시 더해 예측을 원래 스케일로 되돌립니다.

형식적으로, 입력 윈도우 \(x \in \mathbb{R}^{L}\)(한 채널)에 대해 RevIN은 인스턴스 통계량

\[ \mu_x = \frac{1}{L}\sum_{t=1}^{L} x_t, \qquad \sigma_x^2 = \frac{1}{L}\sum_{t=1}^{L}(x_t - \mu_x)^2, \tag{1}\]

을 구하고, 학습 가능한 아핀 변환(매개변수 \(\gamma, \beta\))으로 정규화합니다.

\[ \hat{x}_t = \gamma \, \frac{x_t - \mu_x}{\sqrt{\sigma_x^2 + \epsilon}} + \beta, \tag{2}\]

\(\hat{x}\)를 모델에 통과시켜 정규화된 예측 \(\hat{y}\)를 얻은 뒤, 같은 \(\mu_x, \sigma_x\)로 방정식 2 을 역산하여 역정규화합니다.

\[ y_t = \sqrt{\sigma_x^2 + \epsilon}\;\cdot\;\frac{\hat{y}_t - \beta}{\gamma} + \mu_x. \tag{3}\]

여기서 \(\epsilon\)은 수치 안정을 위한 작은 상수이고, \(\gamma, \beta \in \mathbb{R}\) 는 학습됩니다. 대칭성이 핵심입니다. 각 윈도우의 수준과 스케일이라는 비정상 정보를 입력에서 제거하고 출력에서 복원하므로, 모델은 움직이는 수준이 아니라 정상적인 형태만 학습하면 됩니다 (Kim 기타 2022년).

정확도를 측정하기 전에, 이 정규화가 입력 분포에 무엇을 하는지 직접 보는 것이 좋습니다. 이것이 이 방법의 개념적 핵심입니다.

def windows_only(x, L):
    return np.stack([x[i - L:i] for i in range(L, len(x) - L)])

W = windows_only(series, L)
raw_means = W.mean(axis=1)
raw_stds = W.std(axis=1)

# 인스턴스별 정규화(RevIN의 제거 단계), 아핀 없음:
Wn = (W - W.mean(axis=1, keepdims=True)) / (W.std(axis=1, keepdims=True) + 1e-8)

print("RAW windows    : mean in [%.1f, %.1f], std in [%.1f, %.1f]"
      % (raw_means.min(), raw_means.max(), raw_stds.min(), raw_stds.max()))
print("RevIN-normed   : mean in [%.3f, %.3f], std in [%.3f, %.3f]"
      % (Wn.mean(axis=1).min(), Wn.mean(axis=1).max(),
         Wn.std(axis=1).min(), Wn.std(axis=1).max()))
print("\nWindow means spanned a %.0fx range; RevIN collapses them to ~0."
      % (raw_means.max() / raw_means.min()))
RAW windows    : mean in [1.1, 753.1], std in [0.3, 53.8]
RevIN-normed   : mean in [-0.000, 0.000], std in [1.000, 1.000]

Window means spanned a 702x range; RevIN collapses them to ~0.

원시 윈도우 평균은 수백 대 일의 범위에 걸쳐 있습니다. 인스턴스별 정규화 이후에는 모든 윈도우가 평균 약 0, 표준편차 약 1을 가집니다. 모델의 관점에서 표류하던 시계열이 정상 시계열로 바뀐 것입니다. 이것이 낮은 수준의 초기 데이터로 학습한 모델이 높은 수준의 후기 데이터에서도 여전히 작동하게 만드는 이유입니다. 애초에 수준은 모델이 풀어야 할 문제가 아니었습니다.

5. 측정: 세 전략의 정면 대결

이제 같은 MLP를 세 방식으로 학습합니다. 정규화 없음, 전역 정규화, 그리고 역정규화를 동반한 RevIN 방식 인스턴스 정규화입니다. 각 전략의 정규화가 보이도록 학습 루프를 의도적으로 명시적으로 작성했습니다.

def rmse(a, b):
    return float(np.sqrt(((a - b) ** 2).mean()))

def run(mode, epochs=120):
    torch.manual_seed(0)
    Xt, Yt, Xv = torch.tensor(Xtr), torch.tensor(Ytr), torch.tensor(Xte)
    if mode == "global":
        m, sd = Xt.mean(), Xt.std()
    model = MLP(L, H)
    opt = torch.optim.Adam(model.parameters(), lr=1e-3)
    lossf = nn.MSELoss()
    for _ in range(epochs):
        model.train()
        if mode == "revin":
            mu = Xt.mean(1, keepdim=True); sg = Xt.std(1, keepdim=True) + 1e-5
            xin, ytar = (Xt - mu) / sg, (Yt - mu) / sg   # 입력과 목표를 인스턴스별로 정규화
        elif mode == "global":
            xin, ytar = (Xt - m) / sd, (Yt - m) / sd
        else:  # none
            xin, ytar = Xt, Yt
        opt.zero_grad()
        loss = lossf(model(xin), ytar)
        loss.backward()
        opt.step()
    model.eval()
    with torch.no_grad():
        if mode == "revin":
            mu = Xv.mean(1, keepdim=True); sg = Xv.std(1, keepdim=True) + 1e-5
            pred = (model((Xv - mu) / sg) * sg + mu).numpy()   # 스케일로 역정규화
        elif mode == "global":
            pred = (model((Xv - m) / sd) * sd + m).numpy()
        else:
            pred = model(Xv).numpy()
    return pred

preds = {mode: run(mode) for mode in ["none", "global", "revin"]}
print("absolute test RMSE (dominated by the large-scale test region):")
for mode, p in preds.items():
    print(f"  {mode:8s} {rmse(p, Yte):8.3f}")
absolute test RMSE (dominated by the large-scale test region):
  none       33.316
  global     34.634
  revin      34.112

절대 RMSE 수치는 서로 가깝습니다. 그리고 이 점 자체가 대충 넘기기보다 주의 깊게 읽을 교훈입니다. 두 가지가 일어나고 있습니다. 첫째, 이 테스트 집합의 절대 RMSE는 끝부분의 극히 높은 스케일 윈도우 몇 개에 지배됩니다. 그래서 전략을 비교하기에는 나쁜 렌즈입니다. 큰 스케일 윈도우의 큰 오차가 나머지를 모두 삼켜 버립니다. 둘째, ReLU 층을 가진 평범한 MLP는 그 자체로 스케일을 꽤 학습할 수 있어서, 매끄러운 합성 표류에서는 “정규화 없음”과의 격차가 작습니다. 공정하게 비교하려면 스케일 불변 지표가 필요합니다. 각 윈도우의 오차를 그 윈도우 자신의 수준으로 나누면, 5% 오차는 윈도우가 10에 있든 500에 있든 같은 값으로 계산됩니다.

def relative_rmse(pred, true, Xwin):
    """윈도우별 RMSE를 그 윈도우의 평균 수준으로 나눈 값(MASE와 유사한 스케일 불변 점수)."""
    scale = np.abs(Xwin).mean(axis=1, keepdims=True) + 1e-8
    per_window = np.sqrt(((pred - true) ** 2).mean(axis=1, keepdims=True)) / scale
    return float(per_window.mean())

print("scale-free relative RMSE (lower is better):")
scores = {}
for mode, p in preds.items():
    scores[mode] = relative_rmse(p, Yte, Xte)
    print(f"  {mode:8s} {scores[mode]:.4f}")

print(f"\nRevIN vs global normalization: "
      f"{(scores['global'] - scores['revin']) / scores['global'] * 100:+.1f}%")

  global   0.0829
  revin    0.0804

RevIN vs global normalization: +3.0%

스케일 불변 지표에서 RevIN은 전역 정규화를 몇 퍼센트 차이로 앞서고, 전역 정규화는 여기서 셋 중 실제로 가장 나쁩니다. 자기보다 훨씬 위에 놓인 윈도우들에서 고정 평균 하나를 빼는 것이 아무것도 안 하는 것보다 못한 것입니다. 이 소박하고 정직한 격차가 전형적입니다. 협조적인 모델과 깨끗한 합성 데이터에서는 효과가 실재하지만 작습니다. RevIN에 대한 공개된 근거는 실제 데이터셋과 더 강한 백본 위에서 세워졌으며, 거기서는 트랜스포머나 선형 모델 앞에서 인스턴스별 수준을 제거하는 것이 여러 지평에 걸쳐 상당하고 일관된 이득을 냅니다 (Kim 기타 2022년). RevIN으로 감싼 단순한 선형 모델이 표준 벤치마크에서 훨씬 무거운 모델과 대등하거나 앞선다고 보고되기도 했습니다.

6. 한계 알기: 수준을 제거하는 것이 신호를 제거할 때

RevIN은 공짜가 아니며, 그 이유는 개념적으로 중요합니다. 각 윈도우의 평균과 분산을 벗겨 냄으로써, 당신은 수준과 스케일이 잡음 요인(nuisance) — 제거하고 복원할 정보이지 사용할 정보가 아니다 — 이라고 주장하는 것입니다. 때로 그 주장은 틀립니다. 상승하는 수준이나 분산의 변화 자체가 미래를 예측하는 정보라면, 즉 추세가 정상적인 형태로는 담을 수 없는 방식으로 지평까지 이어진다면, 그것을 정규화로 지워 버리는 것은 신호를 버리는 일입니다.

이것이 Non-stationary Transformer의 비판입니다. 이 연구는 소박한 정상화가 과도한 정상화(over-stationarization)를 일으킬 수 있다고 관찰합니다. 모델이 구별해야 할 바로 그 비정상 사건을 지워 버릴 수 있다는 것입니다. 그래서 저자들은 정상화를, 제거된 통계량을 출력에서만이 아니라 모델 내부로 다시 주입하는 “탈정상화 어텐션(de-stationary attention)”과 짝지었습니다 (Liu 기타 2022년). 이것은 또한 Dish-TS의 동기이기도 합니다. Dish-TS는 RevIN이 입력 윈도우와 지평 윈도우가 통계량을 공유한다고 암묵적으로 가정한다는 점 — 룩백에서 제거한 평균이 지평에서 복원할 올바른 평균이라는 가정 — 을 지적합니다. 입력 공간과 출력 공간의 분포가 정말로 다를 때(공간 간 이동), 통계량 한 벌로는 충분하지 않으며, Dish-TS는 입력 공간과 출력 공간에 대해 별도의 분포 계수를 학습합니다 (Fan 기타 2023년).

실무적 해석은 이렇습니다. 인스턴스 정규화는 흔한 경우(표류하는 수준)를 해결하는 강력하고 값싼 기본값이지만, 하나의 가정 — 수준과 스케일은 잡음 요인이고, 룩백 통계가 지평 통계를 예측한다 — 을 담고 있습니다. 그 가정은 맹목적으로 받아들이지 말고 당신의 데이터에 비추어 확인해야 합니다.

7. 결론

비정상성은 가장자리에서 잡음처럼 제거할 성가신 존재가 아닙니다. 그것은 과거에 맞춘 모델과 미래를 예측하는 모델 사이를 가로막는 중심 장애물입니다. 이 글의 전개는 분야 자체가 걸어온 전개이기도 합니다. 전역 정규화는 최적화를 안정시키지만 움직이는 평균을 따라가지 못하며, 강하게 표류하는 데이터에서는 아무것도 안 하느니만 못할 수 있습니다. 인스턴스별 가역 정규화 — 각 윈도우의 수준과 스케일을 제거하고, 정상적인 형태를 예측한 뒤, 수준을 복원하는 것 — 가 실제로 표류를 다루는 동작이며, 어떤 모델에도 감쌀 만큼 값쌉니다. 그 자신의 한계가 프런티어를 가리킵니다. 수준 자체가 신호일 때, 혹은 룩백과 지평의 분포가 정말로 다를 때는, 그 통계량을 단순히 제거하는 대신 다시 주입하거나 별도로 모델링하는 방법이 필요합니다. 먼저 스케일 불변 지표로 당신의 시계열이 얼마나 표류하는지 측정하고, 그 표류의 크기와 성격이 실제로 얼마나 많은 정규화 장치가 필요한지를 결정하게 하십시오.
